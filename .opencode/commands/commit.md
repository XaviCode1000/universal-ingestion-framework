Create a conventional commit for the UIF project.

## Commit Prefixes

- `feat:` - New feature
- `fix:` - Bug fix
- `refactor:` - Code refactoring
- `test:` - Adding/updating tests
- `docs:` - Documentation changes
- `chore:` - Maintenance tasks
- `perf:` - Performance improvements
- `style:` - Code style changes

## Rules

- Use conventional commits format
- Explain WHY, not WHAT
- Reference AGENTS.md patterns when applicable
- If conflicts exist, DO NOT FIX - notify user
- Keep messages concise (50 chars title max)
